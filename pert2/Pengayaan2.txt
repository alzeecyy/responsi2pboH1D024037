Pertanyaan Refleksi
Setelah menyelesaikan kode, jawab pertanyaan berikut:

Apa yang dimaksud dengan constructor overloading dalam pemrograman berorientasi objek?
Bagaimana cara Java membedakan constructor yang memiliki nama sama namun parameter berbeda?
Apa perbedaan antara mengakses variabel secara langsung (misalnya: objek.variabel = nilai) dengan menggunakan method setter?
Jelaskan konsep encapsulation dan bagaimana method dapat membantu melindungi data dalam sebuah class!
Mengapa constructor tidak memiliki return type, bahkan tidak menggunakan kata kunci void?

Jawaban
1. Kondisi di mana satu class punya banyak konstruktor disebut constructor overloading. Jadi namanya sama, tapi parameter yang dipakai harus berbeda.
2. Java akan otomatis milih konstruktor yang paling cocok berdasarkan nama, jumlah parameter, dan tipe data yang dikirim.
3. Getter biasanya dipakai buat ngambil nilai variabel, sedangkan setter dipakai buat mengubah nilai variabel, khususnya yang dibuat private supaya tetap terjaga prinsip enkapsulasi.
4. Enkapsulasi adalah kondisi di mana atribut atau method dibungkus dengan akses private, sehingga tidak bisa diakses langsung dari luar class kecuali lewat method getter dan setter.
5. Konstruktor tidak boleh punya return karena kalau ada return, Java akan menganggapnya sebagai method biasa. Selain itu, konstruktor otomatis dipanggil saat objek baru dibuat.
